# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ShineTwoPlay, please **do not** open a public issue.

Instead, please report it responsibly by contacting the maintainer directly:

- **GitHub**: [@nikh27](https://github.com/nikh27) — send a private message or use [GitHub Security Advisories](https://github.com/nikh27/shinetwoplay/security/advisories/new)

### What to include in your report

1. **Description** of the vulnerability
2. **Steps to reproduce** the issue
3. **Potential impact** (what could an attacker do?)
4. **Suggested fix** (if you have one)

### What to expect

- You will receive an acknowledgment within **48 hours**
- The maintainer will work on a fix and coordinate a release
- You will be credited in the release notes (unless you prefer to remain anonymous)

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest `main` branch | ✅ |
| Older commits | ❌ |

## Best Practices

- Never commit secrets, API keys, or passwords to the repository
- Always use environment variables for sensitive configuration (see `.env.example`)
- Keep dependencies up to date

Thank you for helping keep ShineTwoPlay safe! 🔒
